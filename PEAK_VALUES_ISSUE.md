# Проблема с пиковыми значениями (Peak Players)

## Текущая ситуация

**Проблема:** SteamCharts API (`chart-data.json`) возвращает только **средние значения** игроков. Пиковые значения доступны только на HTML странице в таблице.

**Текущий код:** Сохраняет одно и то же значение как `avg` и `peak` (строки 200-210 в `steamcharts_parser.py`), что неправильно.

## Решение

Нужно добавить парсинг HTML страницы SteamCharts для получения пиковых значений из таблицы.

### Вариант 1: Парсинг HTML страницы (рекомендуется)

Добавить метод для парсинга HTML страницы SteamCharts и извлечения peak значений из таблицы.

**Структура HTML таблицы:**
```html
<table>
  <tr>
    <th>Month</th>
    <th>Avg. Players</th>
    <th>Peak Players</th>
  </tr>
  <tr>
    <td>2024-01</td>
    <td>1000</td>
    <td>1500</td>
  </tr>
</table>
```

### Вариант 2: Использовать другой источник данных

Если есть альтернативный API или источник данных с отдельными avg и peak значениями.

## Текущий статус данных

- ✅ Средние значения (avg) собираются правильно из API
- ❌ Пиковые значения (peak) дублируют средние (неправильно)
- ✅ Формат экспорта исправлен: `ID,datetime,avg_players,peak_players`

## Что нужно сделать

1. Добавить парсинг HTML страницы SteamCharts для получения peak значений
2. Обновить `steamcharts_parser.py` для правильного сохранения avg и peak
3. Перезапустить парсинг для сбора правильных peak значений

## Приоритет

**КРИТИЧЕСКИЙ** - Пиковые значения являются наиболее важным критерием для пользователя.
